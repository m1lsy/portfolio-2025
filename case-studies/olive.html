Olive Recipe App

Role

UX Designer & Flutter Developer

Timeline

3 weeks, 2025

Tools

Flutter, Figma, TheMealDB API

Overview

Olive is a mobile app that helps students discover and save recipes through an intuitive, swipe-based interface. The project addresses the common challenge of meal decision fatigue among students while providing a streamlined way to collect cooking inspiration.

How might we help students easily collect, save, and access meal inspiration through a mobile experience?

Challenge

As a student myself, I often struggle with deciding what to cook every day. Many students face similar challenges:

  • Decision fatigue when choosing meals
  • Difficulty finding quick, affordable, and nutritious recipes
  • Scattered recipe inspiration across various social platforms
  • Lack of organization for saved recipes
  • Incomplete recipe information on social media

User Research

Interview Insights

I conducted interviews with two university students to understand their cooking habits and recipe discovery processes:

  • Luca (23, Italy) – Cooks 4-5 times weekly, prioritizes affordable, flavorful meals
  • Veele (21, Netherlands) – Daily cook, focuses on healthy, low-effort meals

Key Findings

  1. Decision Fatigue: Students spend excessive time choosing meals, often defaulting to familiar options.
  2. Fragmented Discovery: Existing apps are paywalled or ad-heavy; social media recipes lack complete information.
  3. Speed Priority: 30-minute meals using available ingredients are preferred.
  4. Swipe Appeal: The "Tinder-for-recipes" concept received positive feedback for its playful, efficient approach.
  5. Visual Focus: Clear imagery and minimal text are essential for quick decision-making.

Design Process

Low-Fidelity Prototypes

I started with black and white wireframes to explore the basic structure and user flows, focusing on core interactions like recipe cards and swipe gestures.

Visual Design Evolution

The high-fidelity design process involved careful consideration of:

  • A calm neutral palette with green accents reflecting the Olive brand
  • Typography combining Astonpoliz for branding and Inter for readability
  • Minimalist interface focusing on food imagery
  • Refined UI elements based on peer feedback

Design Validation

User testing with students confirmed our design decisions:

  • Green and muted red accents improved visual hierarchy
  • Revised typography enhanced readability
  • Right-aligned heart button matched user expectations
  • Clean border radius treatment modernized the interface

Technical Implementation

Architecture & Technologies

Olive was built using Flutter for cross-platform performance, following clean architecture principles:

  • TheMealDB API integration for dynamic recipe content
  • Hive NoSQL database for offline recipe storage
  • Repository pattern separating data and UI layers
  • Singleton SavedStore with ChangeNotifier for state management

Key Features

The app implements several technical features to create a seamless experience:

  • Reusable widgets (RecipeCard, HomeActionBar, TagChip)
  • Animated card transitions and visual feedback
  • Offline support for saved recipes
  • Proper error handling and loading states

Collaborative Development

The project utilized GitLab for version control, implementing:

  • Structured branching strategy for feature development
  • Peer review process to maintain code quality
  • AI-assisted work distribution between team members
  • Regular integration checkpoints for feature validation

Outcomes

The project delivered several key achievements:

  • Successfully implemented a Flutter-based recipe discovery app
  • Created an intuitive, swipe-based interface for recipe browsing
  • Integrated offline support for saved recipes
  • Received positive feedback from student user testing
The swipe-based interface makes finding recipes fun and easy. It's like Tinder for food! — Student Tester

Learning Through Constraints

The three-week timeline for this project pushed me to adapt my typically thorough research and iteration process into a focused sprint. This constraint became a valuable learning opportunity:

  • Practiced rapid decision-making and prioritization of features
  • Learned to conduct efficient user research with a smaller but focused participant group
  • Developed skills in quick prototyping and iterative feedback loops
  • Gained experience in adapting design processes to tight deadlines while maintaining quality

This experience taught me how to be more adaptable and efficient in my design process while still delivering meaningful solutions. The compressed timeline helped me develop a more streamlined approach to user research and iteration, skills that will be valuable in future professional projects.
